SIGEANSUD

AP 5 · Gestion des repas du parc animalier · Symfony 7 + Android + API REST

Contexte du projet

Le projet SIGEANSUD consiste à développer un applicatif complet pour gérer les repas des animaux du parc animalier SIGEANSUD. L'objectif est de tracer tous les repas donnés aux animaux et de gérer les menus par espèce, avec deux interfaces distinctes : une application mobile pour les soignants et un back-office web pour les gestionnaires.

Réalisé en équipe de quatre (Mateo, Samuel, Clayton et Ewen) selon une approche Agile avec Trello, le projet a été déployé sur serveur distant IONOS.

Applications développées

  • Back-office web (Symfony 7) — interface d'administration pour les gestionnaires : gestion des soignants et de leurs espèces affectées, gestion des menus par espèce, consultation des repas tracés · tableau de bord avec stats (soignants, espèces, menus)
  • Application mobile Android (Java) — interface pour les soignants : saisie des heures et quantités des repas donnés aux animaux des espèces affectées · stockage local SQLite du profil soignant
  • API REST (MySQL) — backend commun aux deux applications : URIs pour accéder aux données depuis le front web et l'app mobile · déployée sur IONOS avec base MySQL

Ma contribution

Avec Matéo, j'ai développé l'application mobile Android destinée aux soignants du parc : saisie des repas, consultation des animaux par espèce, visualisation des menus du jour et historique des repas donnés.

Application mobile Android Interface soignant – SIGEANSUD

Écran d'accueil de l'application Android : menu principal permettant au soignant de saisir un repas, consulter les animaux par espèce affectée, voir les menus du jour et accéder à l'historique des repas. Développé en Java avec Android Studio, connecté à l'API REST via MySQL IONOS.

Architecture technique

  • Symfony 7 — back-office web, déployé sur IONOS
  • Android Studio / Java — application mobile pour smartphones
  • API REST — point d'entrée unique pour le web et le mobile, spécifiée et documentée
  • MySQL (IONOS) — base de données distante partagée via l'API
  • SQLite — base locale sur l'app mobile pour stocker le profil soignant
  • MCD + diagramme de classes — modélisation complète des données
  • MOT / diagramme de séquence — modélisation des flux entre front, back et API

Gestion de projet

  • Méthode Agile — planification par sprints, répartition des tâches par étudiant
  • Trello — suivi de l'avancement et collaboration

Gestion des triathlons

AP 4 · Application web FFTRI · Architecture MVC

Présentation du projet

Dans le cadre de l'AP 4, notre équipe a développé une application web complète de gestion pour la FFTRI (Fédération Française de Triathlon). L'objectif était de centraliser la gestion des clubs affiliés, des licenciés et des triathlons à venir au sein d'une interface d'administration sécurisée.

L'application repose sur une architecture MVC (Modèle-Vue-Contrôleur) en PHP natif, avec une gestion de version via GitHub et un suivi de projet sur Trello. Le projet a été réalisé en équipe : Samuel Aubron, Noah Herpin, Yasmin Khanjaoui et Matéo.

Modules de l'application

  • Tableau de bord — vue d'ensemble avec statistiques en temps réel : clubs affiliés, licenciés actifs, triathlons à venir · affichage des derniers clubs ajoutés
  • Clubs — CRUD complet : liste, fiche détaillée, création, modification, suppression · informations : nom, ville, téléphone, licenciés
  • Licenciés — gestion des licenciés avec 2 types de licence : club ou individuelle
  • Triathlons — gestion des épreuves : nom, type, lieu, date · organisation des inscriptions · 2 profils : Responsable de club et Gestionnaire FFTRI
  • Paramètres — mise à jour du profil, changement de mot de passe, préférences et notifications

Ma contribution

J'ai pris en charge deux réalisations clés : la conception de la maquette graphique de l'application et le développement de la page de gestion des clubs avec son CRUD complet.

Design UI Maquette de l'application

Conception de la maquette graphique de l'interface d'administration FFTRI : sidebar de navigation, tableau de bord et gabarit des pages avant développement.

Développement PHP MVC Page Gestion des clubs

Développement de la page listant les clubs affiliés (nom, ville, téléphone, email, nombre de licenciés) avec actions d'édition et de suppression. CRUD complet intégré dans l'architecture MVC.

Architecture technique

  • Pattern MVC — séparation stricte Modèle / Vue / Contrôleur pour un code maintenable
  • Routage — système de routing via paramètres GET (module + action) : /index.php?module=clubs&action=create
  • Authentification — accès sécurisé obligatoire, sessions PHP, routes publiques limitées à login/logout
  • Base de données — MySQL avec requêtes préparées
  • Interface — sidebar de navigation fixe, design professionnel responsive, maquettes réalisées sous Canva

Routes de l'application

  • Authentification — /index.php?action=login · authenticate · logout
  • Dashboard — /index.php?module=dashboard
  • Clubs — module=clubs + actions : show, create, edit, delete
  • Licenciés — module=licencies
  • Triathlons — module=triathlons
  • Paramètres — module=settings + updateProfile, changePassword, updatePreferences
